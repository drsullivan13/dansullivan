# Portfolio Arcade - Coding Agent Instructions

## Vision

Transform a portfolio website into an **interactive Galaga-inspired experience** where visitors pilot a ship through your career, shooting at sections to explore projects, blog posts, and recipes. But here's the critical insight: *The game IS the interface, not a gimmick blocking content.*

**Design Philosophy:** "Advanced technology should feel like magic, not a maze." - Make it delightful, instant, and optional.

## Mechanics (Keep It Simple)

### Player Ship
- **Position:** Bottom third of screen, moves horizontally only
- **Controls:** 
  - Desktop: Arrow keys (â†â†’) or A/D to move, SPACE to shoot
  - Mobile: Touch left/right side of screen to move, tap ship to shoot
  - Alternative: Click/tap anywhere to move ship there smoothly
- **Visual:** Retro pixel-art style ship, subtle engine glow
- **Speed:** Responsive but not twitchy (300px/sec)

### Target Sections (Top Formation)
- **Layout:** 4 "targets" arranged like Galaga enemies
  - **PROJECTS** (left) - 12 hits to destroy
  - **BLOG** (center-left) - 8 hits
  - **RECIPES** (center-right) - 5 hits  
  - **ABOUT** (right) - 1 hit (instant access)
- **Visual:** Each target is a rounded rectangle with icon + label + HP bar
- **Behavior:** 
  - Gentle bobbing animation (like floating)
  - Pulse/glow on hit
  - Health bar depletes with each shot
  - "Explode" animation on destruction â†’ Navigate to that section
- **Hit Points = Content Volume** (Projects has most content, About is quick)

### Projectiles
- **Visual:** Simple colored bullets (matching accent color)
- **Speed:** Fast enough to feel responsive (800px/sec)
- **Collision:** Detect hit, reduce HP, visual feedback
- **Sound:** Optional subtle "pew" and "hit" sounds (can be muted)

### Background
- **Starfield:** Parallax scrolling stars (3 layers, different speeds)
- **Color:** Deep space gradient (dark blue â†’ black)
- **Performance:** Canvas-based for smooth 60fps

## Detailed Requirements

### Landing Experience

**First Load:**
1. Starfield fades in (500ms)
2. Targets materialize from top with stagger (200ms each)
3. Player ship rises from bottom (300ms)
4. Tutorial overlay appears (if first visit)
5. Game becomes active

**Total load time:** < 1.5 seconds

**Hit Feedback:**

4. Fade to section page (300ms transition)
5. Confetti/celebration effect (brief)

### Projectile Physics

**Bullet:**
- Size: 4px Ã— 12px (small, fast)
- Color: Accent color with glow
- Speed: 800px/sec straight up
- Max on screen: 3 bullets (prevents spam)
- Collision: Simple bounding box (fast calculation)

**Rate of Fire:**
- Cooldown: 300ms between shots
- Visual: Ship glows during cooldown (can't shoot)
- Feel: Deliberate, not spray-and-pray

### Starfield Animation

**Three Layers:**
- Layer 1 (far): 50 small stars, slow (20px/sec)
- Layer 2 (mid): 30 medium stars, medium (40px/sec)  
- Layer 3 (near): 20 large stars, fast (80px/sec)

**Visual:**
- White dots with slight glow
- Twinkling opacity variation
- Wrap at bottom (infinite scroll)
- Parallax creates depth

### Score System

**Points:**
- Hit any target: +10
- Destroy ABOUT: +100 (easiest)
- Destroy RECIPES: +500
- Destroy BLOG: +800
- Destroy PROJECTS: +1200 (hardest)

**Display:**
- Top-left corner
- Large, retro font
- Animates on increase
- High score persisted (localStorage)

## Success Criteria

### The Game Should Feel:
- **Responsive:** No input lag, buttery smooth
- **Rewarding:** Satisfying feedback on every hit
- **Optional:** Never blocks access to content
- **Polished:** Pixel-perfect animations, no jank
- **Fast:** Loads instantly, runs at 60fps

### User Testing Questions:
1. "Did you smile when you first saw it?"
2. "Could you access the portfolio without frustration?"
3. "Did the game enhance or distract from the content?"
4. "Would you share this with a colleague?"

**If any answer is "no," iterate.**
